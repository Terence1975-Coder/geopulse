Configuration Workspace Guide

This guide explains the GeoPulse Calibration & Settings Workspace so a user can understand what each section controls, why it matters, and how those settings influence platform behaviour.

Module: Configuration
Audience: End users / pilot users / admins
Purpose: Platform calibration and control
Overview

What this workspace does

The Configuration workspace is the control surface for tuning how GeoPulse interprets signals, how assertive the agents should be, how often sources refresh, and how privacy-related permissions are handled.

Why a user would use it

  • Adjust how sensitive GeoPulse is to incoming signals.
  • Shift the platform toward risk detection or upside detection.
  • Control tone and posture of embedded agents.
  • Enable or restrict live ingestion and external enrichment.
  • Set privacy boundaries for anonymisation and profile enrichment.

What this changes in practice

  • What kinds of signals are surfaced sooner or filtered out.
  • How firm or cautious recommendations feel.
  • Whether the system leans short-term or longer-term.
  • How much external context GeoPulse is allowed to use.
  • Whether user data is masked before deeper AI processing.
Screen layout

How the page is organised

The workspace is arranged as a set of configuration cards. Each card groups related controls so the user can quickly tune one part of the system without hunting through unrelated settings.

Area Purpose User outcome
Intelligence Settings Controls how GeoPulse scores and prioritises signals. Changes the balance between alert filtering, severity focus, and opportunity sensitivity.
Agent Behaviour Settings Controls how the Analyst, Advisor, and Profile-related behaviours feel. Changes tone, stance, and how assertive recommendations become.
Source Settings Controls whether ingestion is active and how often data refreshes. Changes the freshness and reach of live intelligence inputs.
Privacy Settings Controls masking and enrichment permissions. Changes what data can be processed, enriched, or protected.
Section 1

Intelligence Settings

These controls shape the way GeoPulse interprets incoming intelligence and how strongly different factors influence scoring.

Control What it does How a user should think about it
Signal Threshold Sets the minimum signal strength needed before GeoPulse treats something as important enough to surface. Lower values make GeoPulse more sensitive. Higher values make it more selective.
Severity Weighting Increases or reduces the emphasis placed on the seriousness of a signal. Raise this when the user wants the platform to prioritise high-impact disruptions.
Opportunity Sensitivity Controls how readily positive or growth-related openings are detected. Raise this when the user wants more upside discovery, not just threat monitoring.
Risk / Opportunity Weighting Balances the platform between defensive monitoring and proactive upside detection. Move lower to lean toward risk control, or higher to lean toward opportunity capture, depending on how your scoring model is defined in deployment.
Horizon Preference Sets the preferred time lens for the system: immediate, balanced, or longer-term. Useful when a user wants GeoPulse focused on urgent issues versus emerging medium-term developments.
Best-use guidance: If a client wants earlier warning signals, lower the threshold slightly and keep severity weighting moderately high. If they want more growth ideas, increase opportunity sensitivity and keep the horizon balanced or longer-term.
Section 2

Agent Behaviour Settings

These controls shape how GeoPulse agents communicate and how forcefully recommendations are framed.

Control What it does User effect
Analyst Tone Mode Sets the style of analytical output, such as boardroom, analytical, or calm. Changes the feel of analysis without changing the underlying intelligence itself.
Advisor Stance Mode Sets whether the advisor behaves more conservatively, in a balanced way, or more aggressively. Useful for matching the user’s decision style or company risk appetite.
Recommendation Aggressiveness Controls how assertive GeoPulse should be when suggesting actions. Higher values create stronger, more forward-leaning recommendations.
Profile Question Depth Controls how deeply GeoPulse should explore company-specific profile information. Deeper settings can improve relevance, but may require more user input.
Best-use guidance: For executive users, a boardroom or analytical tone with balanced stance usually works well. For internal testing, deeper profile questioning can help improve calibration quality faster.
Section 3

Source Settings

These controls determine how live the platform is and how much external data it is allowed to bring into its reasoning environment.

Control What it does User effect
Live Ingestion Enabled Turns the live signal ingestion layer on or off. When enabled, GeoPulse can work with fresher intelligence inputs.
Refresh Cadence Sets how often the system should refresh or pull new data. Shorter cadence improves freshness but may increase noise and system load.
External Lookup Enabled Allows GeoPulse to use external lookup or enrichment sources where supported. Improves context depth, but should be controlled carefully for privacy and trust reasons.
Section 4

Privacy Settings

These controls define what GeoPulse is permitted to do with user and company data before it enriches, interprets, or passes information through AI workflows.

Control What it does User effect
Anonymisation Enabled Masks or redacts sensitive data before deeper processing. Improves privacy protection and is especially valuable for enterprise or sensitive company inputs.
Profile Enrichment Allowed Determines whether GeoPulse is allowed to enrich the company profile using available context or lookup sources. Can improve relevance, but should only be enabled when the user is comfortable with that level of processing.
Best-use guidance: For pilots and enterprise demonstrations, it usually makes sense to keep anonymisation enabled by default, then selectively allow enrichment where the client has approved it.
How to use it

Suggested user workflow

  1. Open the Configuration workspace from the main navigation.
  2. Review Intelligence Settings first to make sure signal scoring matches the user’s current priorities.
  3. Adjust Agent Behaviour so the Analyst and Advisor match the client’s preferred tone and level of assertiveness.
  4. Confirm Source Settings if live monitoring is required for the current session or pilot.
  5. Review Privacy Settings before using sensitive company data or profile enrichment.
  6. Return to other workspaces such as Executive Dashboard, Analyst, Advisor, or Company Intelligence to see the operational impact of the chosen settings.
User notes

Important interpretation notes

What this page is today

In the current build, this workspace is a front-end control surface. It provides live UI controls for calibration and settings, and is positioned as the central place for tuning GeoPulse behaviour.

What this page is expected to become

As GeoPulse matures, these controls can be connected to persistence, backend calibration logic, policy enforcement, and role-based administration.
